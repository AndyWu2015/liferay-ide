Creating New Liferay Plug-in Projects

  1. Liferay IDE supports development of five Liferay plug-in types: portlets, hooks, layout templates, themes, and EXT-style plug-ins.
  2. To create a new plug-in project with Liferay IDE, select the Liferay Plug-in Project option from the Eclipse File menu. (Go to File > New > Liferay Plug-in Project).
  3. Choose a project name for your plugin. The project name will serve as the file name of your project. Note that the project name cannot include any blank spaces.
  4. Choose a display name for your plugin. The display name will serve as the title of your project and can include blank spaces.
  5. Under Configuration, you will need to select a Plug-ins SDK and a Runtime. You should already have these installed if you followed the directions in the Getting Started section and can simply select them from a list. As defaults, you should use the latest Liferay Plug-ins SDK and the latest Liferay Portal Tomcat Server.
  6. Under Plug-in Type, select Portlet.
  7. Click Finish, and your project should automatically build for you. You can view and modify your project using the Package Explorer that sits at the left side of the screen when using the Liferay perspective within Eclipse.

Alternatively, you can create a new portlet, hook, service, or layout template project by making the appropriate selection from the File menu of Eclipse.

Creating New Portlets

  1. You can create a new portlet project with Liferay IDE by selecting Liferay Portlet from the Eclipse File menu (File > New > Liferay Portlet).
  2. You will input the information required to create a portlet class on this screen.
  3. Under Portlet plug-in project, choose a name for your portlet.
  4. The Source folder determines the location of the file directory for your portlet project.
  5. Portet class specifies the name of the Java class that your portlet extends: NewPortlet.
  6. Java package specifies the location of the Java objects that NewPortlet needs to use.
  7. Superclass specifies the location of the location of the MVCPortlet object which NewPortlet extends. Click next.
  9. You will specify portlet deployment descriptor details on this screen.
  10. Under Portlet Info, enter the name, display name, and title of your portlet.
  11. Under Portlet Modes, you can choose whether or not you want your portlet to have view, edit, and display modes by checking or unchecking the appropriate boxes.
  12. Under Liferay Portlet Modes, you can choose which of six modes available to Liferay portlets you want your portlet to use.
  13. Under Resources, you can decide whether or not your portlet will create JSP files. If your portlet will do so, you have to specify a JSP folder. Additionally, you can decide whether or not your portlet will use a resource bundle file. If it will, then you have to specify the resource bundle file path. Click next.
  15. You will continue specifying portlet deployment descriptor details on this screen.
  16. Under Liferay Portlet Info you can name your portlet and associate an icon to it. Check the Allow multiple instances button if you want your portlet to be instanceable. If your portlet will use CSS and Javascript you need to specify the locations of your portlet project’s CSS and Javascript folder and give your portlet a CSS classname.
  17. Under Liferay Display you can select a display category (default category: sample). There is a long list to choose from. Click next.

  1. On this last page you can specify modifiers, interfaces, and method stubs to generate within your portlet class.
  2. For modifiers, you can choose public, abstract, or final.
  3. You can add an interface by specifying its location. Click add, then browse to its location, then select.
  4. If you would like to inherit constructors or abstract methods from your portlet’s superclass, check the appropriate boxes. Then you can select the specific methods you would like to use for your portlet class.
  5. When you are done, click Finish, and your project should automatically build for you.
